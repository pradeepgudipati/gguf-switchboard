# gguf-switchboard v0.1.1

**An OpenAI-compatible GGUF runtime** — a llama-swap-style proxy in Rust that loads and swaps `llama-server` backends on demand.

## Highlights

- Single OpenAI-compatible endpoint (`/v1/chat/completions`, `/v1/completions`, `/v1/embeddings`, `/v1/responses`, `/v1/audio/*`, `/v1/models`)
- On-demand model swap with idle timeout and priority model auto-load
- VRAM-pressure eviction and context-size fallback on OOM
- Built-in usage history at `/v1/usage` (SQLite)
- Swagger UI at `/swagger-ui/`
- `deploy.sh` for Linux systemd install from the `main` branch

## Install

### Prebuilt binary (Linux)

```bash
curl -fsSL -o gguf-switchboard \
  https://github.com/pradeepgudipati/gguf-switchboard/releases/download/v0.1.1/gguf-switchboard-linux-amd64
chmod +x gguf-switchboard
```

See **Assets** below for `linux-arm64`, macOS builds, and `checksums.txt`.

### From source

```bash
git clone --branch main https://github.com/pradeepgudipati/gguf-switchboard.git
cd gguf-switchboard
./deploy.sh
```

## Requirements

- Linux (recommended) or macOS for manual builds
- [llama.cpp](https://github.com/ggerganov/llama.cpp) `llama-server` on your `PATH` (CUDA or Metal)
- At least one GGUF model file
