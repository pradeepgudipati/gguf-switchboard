# gguf-switchboard v0.1.2

**llama-swap alternative in Rust** with memory-pressure eviction, OOM context fallback, Swagger UI, and built-in usage tracking.

## What's new

### Model registry

- Split model definitions into `models.toml` (separate from `config.toml`)
- `discover-models` CLI — auto-generate registry from a GGUF directory
- `export-registry` CLI and `GET /v1/models/registry.json` for portable `models.json`
- Per-model fields: `kind`, `enabled`, `extra_args`, deduplication, single `priority` enforcement
- `deploy.sh` syncs `/etc/gguf-switchboard/models.toml` to gitignored `models.local.toml` / `models.local.json`

### GPU-aware defaults

- `vram_gb` in `config.toml` sizes per-model `-c` from file size and model kind
- Default fallback context lowered to **16384** (safer on consumer GPUs; raise in `models.toml` if you have headroom)
- Multi-directory GGUF discovery (`models_dir` comma-separated paths)

### Docs & UX

- README reframed as an honest [llama-swap](https://github.com/mostlygeek/llama-swap) alternative with a tightened landscape comparison
- Prebuilt-binary quick start from `main` / Releases
- Swagger UI: fixed editable payload reset; models.json download link in the top bar

## Install

### Prebuilt binary (Linux)

```bash
curl -fsSL -o gguf-switchboard \
  https://github.com/pradeepgudipati/gguf-switchboard/releases/download/v0.1.2/gguf-switchboard-linux-amd64
chmod +x gguf-switchboard
```

See **Assets** for `linux-arm64`, macOS builds, and `checksums.txt`.

### Upgrade from v0.1.1

```bash
cd gguf-switchboard
git checkout main && git pull
./deploy.sh
```

Existing `config.toml` is preserved. On first run after upgrade, review `models.toml` under your config dir (default `/etc/gguf-switchboard/`).
